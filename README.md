# 회의 잡기 — Toss PD Challenge 2026

6명이 다음 주까지 1시간 회의를 잡는 경험. 핵심 문제 정의:
**일정 조율의 진짜 고통은 빈 시간 찾기가 아니라, 모두가 안 될 때 '누굴 포기할지'를 주최자가 혼자·미안하게 떠안는 것.**

이 저장소는 히어로 화면(단일 추천 + 거래 제안)의 인터랙티브 프로토타입입니다.

## 실행

```bash
# 로컬에서 열기 — 빌드 없음, 정적 파일
open index.html
# 또는 간단한 서버
python3 -m http.server 8000
```

## 해상도 결정

- **설계 해상도: 375×812 모바일 네이티브.** 일정 조율은 이동 중·메신저 옆에서 일어나는 모바일 과업이고, TDS는 모바일 컴포넌트 시스템.
- **제출 형태: 반응형 프레젠테이션 셸.** 심사위원은 데스크톱에서 URL을 열기 때문에, ≥768px에서는 폰 프레임 + 설계 근거 패널로 렌더링. <768px에서는 풀블리드 네이티브.

## 구조

```
index.html          히어로 화면 (추천 + 거래)
css/tokens.css      TDS 문법 디자인 토큰 (피그마 Variables 원본)
css/screen.css      화면 스타일 — 5단 타이포 위계 · 강조 리듬 · 모션
js/interactions.js  마이크로 인터랙션
docs/FIGMA_TOKENS.md  피그마 이식 가이드
```

## 조형 원칙 3줄

1. **타이포 5단 위계** — Display 26/800 ~ Micro 11/600. 시간(결정 대상)이 화면에서 가장 크다.
2. **강조 리듬 3단** — 히어로(elevation) > 거래(blue 틴트) > 차선(flat). 한 화면에 그림자 카드는 하나.
3. **모션은 의미 있는 곳에만** — 아바타 스태거 팝(가능 인원이 '차오르는' 느낌), 카드 라이즈. `prefers-reduced-motion` 존중.
