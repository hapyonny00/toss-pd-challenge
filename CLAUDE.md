# CLAUDE.md — 작업 컨텍스트

Claude Code가 이 저장소에서 작업할 때 참고하는 문서입니다.

## 이 프로젝트가 뭔가

토스 2026 프로덕트 디자인 챌린지 제출물. 과제: **같은 회사 동료 6명이 다음 주까지 1시간 회의를 잡는 경험 설계.**

핵심 문제 정의(모든 결정의 기준점):
> 일정 조율의 진짜 고통은 빈 시간 찾기가 아니라, 모두가 안 될 때 **'누굴 포기할지'를 주최자가 혼자·미안하게 떠안는 것**이다. 시스템이 제약의 무게를 흡수해, 주최자가 판단도 죄책감도 지지 않게 한다.

## 편집 규칙

- **`index.html` 단 하나만 편집한다.** 빌드 없음. 브라우저로 바로 열림.
- `prototype.html`은 v1 모바일 네이티브 아카이브(참고용). 건드리지 않는다.
- 화면은 `<main class="page" id="p-xxx">`. id로 지시하면 정확히 찾아짐.
- 색·타이포·여백은 `:root`의 **WDS 시맨틱 토큰**만 쓴다. 하드코딩 금지.
  (primary `#0066FF` · coolNeutral 램프 · label/line은 알파 토큰. 토스 블루 `#0064FF`는 로고 전용)
- 폰트: Wanted Sans Variable(CDN) → Pretendard 폴백.
- 반응형: <768 모바일 풀블리드 / ≥1024 본문 640 + 설계 노트 레일 300.
- 설계 노트(데스크톱 레일)는 script의 `RATIONALE` 객체. 화면을 바꾸면 노트도 같이 갱신.

## 절대 지켜야 할 설계 원칙 (바꾸지 말 것)

1. **막다른 길 금지.** 모든 화면에 출구가 있어야 함(추천 다 별로 → 거래, 동료 다 안 됨 → 되는 시간 입력, 대기 → 5명 확정).
2. **묻지 말고 아는 걸 쓴다.** 캘린더 free/busy는 자동, 캘린더가 모르는 선호만 탭으로.
3. **One Thing per One Page.** 한 화면에 주 결정 하나. 주 버튼(파랑)은 화면당 하나.
4. **강조 리듬 3단.** 히어로(그림자) > 거래(파랑 틴트) > 차선(flat). 그림자 카드는 화면당 하나.
5. **라이팅은 토스 원칙대로.** 해요체·능동·긍정·캐주얼 경어('~시' 제거, '께'→'에게').
   `docs/UX_WRITING.md`의 금지어(가중치·최적화·교집합) 쓰지 말 것.
6. **확정 = 등록.** 캘린더 등록은 '과정'이 아니라 자동으로 끝난 '결과'로 보여준다.
   개인 캘린더 내보내기(Google/Outlook/ICS)는 실동작을 유지한다.

## 캘린더 엔진 (script 하단)

- 다음 주 월–금을 오늘 기준으로 동적 계산 (`nextWeekday`). 목 10시 확정 / 화 11시 변경.
- `data-when="thu-short|thu-long|tue-long|range|..."` 속성에 날짜 텍스트 주입.
- Google Calendar URL 템플릿 / Outlook deeplink / ICS Blob 다운로드(VALARM 포함).
- 알림 칩(10/30/60분)은 `alarmMin` 상태 → ICS TRIGGER에 반영.
- 주간 뷰는 `renderWeek()`가 그린다: 9–18시, 1시간=16px, active/ghost/gone 블록.
