# UX 라이팅 시스템

토스 공식 UX 라이팅 원칙(developers-apps-in-toss.toss.im/design/ux-writing)을
이 제품에 그대로 적용하고, 제품 고유의 원칙 5개를 더했습니다.

## 토스 공식 원칙 → 적용

| 토스 원칙 | 이 제품에서 |
|---|---|
| **해요체** | 전 화면 해요체 통일. "넣었어요 / 바꿨어요 / 돼요" |
| **능동적 말하기** | "등록됐어요" ✕ → "넣었어요" ✓. "확인되었습니다" ✕ → "확인했어요" ✓ |
| **긍정적 말하기** | "안 돼요" 대신 출구 제시 — "안 옮겨도 5명으로 그대로 진행돼요" |
| **캐주얼한 경어** | '~시' 제거: "오셔야 하는" → "와야 하는". '께' → '에게': "박지후님에게" |
| **{명사}+{명사} 회피** | "응답 현황" → "답한 사람", "선호 응답" → "답했어요" |

예외 규칙도 따릅니다 — 동료에게 보내는 **대필 메시지**는 제품 문구가 아니라
사람 사이의 말이라, 자연스러운 존댓말("옮겨주실 수 있을까요?")을 유지해요.

## 제품 고유 원칙 5

1. **버튼은 결과를 미리 말한다** — "확인" 대신 "추천 시간 보기", "목요일 오전 10시로 잡기".
2. **행동의 결과를 즉시 되돌려준다** — 보내기 → 토스트 "답을 보냈어요".
3. **시스템 용어 대신 사람 말** — 필수/선택 → "꼭 와야 해요 / 안 와도 돼요".
4. **누구의 잘못도 아니게** — "불참 ❌"이 아니라 "선택 참석이라, 외근이어도 회의는 그대로 진행돼요".
5. **과장하지 않는다** — "박지후님이 조정하면 6명 전원이 모여요". 조건은 조건문으로.

## 화면별 적용

| 화면 | 문구 | 원칙 |
|---|---|---|
| 홈 | "확정하면 모두의 캘린더에 자동으로 들어가요" | 결과 예고 |
| 만들기 | "누가 모여야 하나요?" (기능명 아닌 질문) | 사람 말 |
| 만들기 | "나(김화연)는 자동으로 들어가요" | 능동 |
| 조율 | "일정 제목은 안 보고, 비어 있는지만 봤어요" | 안심(예외: 수동 아님, 능동 유지) |
| 동료 탭 | "안 되는 시간만 알려주세요 · 다 괜찮으면 그대로 보내요" | 결과 예고 |
| 추천 | "이 시간이 가장 좋아요" + 근거 한 줄 | 과장 없음 |
| 조정 부탁 | "어려우면 목요일 10시로 진행할게요!" (상대에게 출구) | 긍정 |
| 대기 | "급하지 않으면 그냥 기다려도 괜찮아요" | 행동 강요 없음 |
| 확정 | "6명 캘린더에 자동으로 넣었어요 · 다들 알림을 받았어요" | 능동·즉시 피드백 |
| 캘린더 담기 | "개인 캘린더에도 담을래요?" ('추가하시겠어요?' ✕) | 캐주얼 경어 |
| 캘린더 담기 | "아까 담았어도 괜찮아요. 새 시간으로 바꿔서 담겨요" | 안심 |
| 변경 | "4명이 와요 · 지금보다 1명 줄어요" (손해도 정직하게) | 과장 없음 |

## 금지어

계산·시스템 냄새가 나는 말: ~~가중치, 최적화, 교집합, 알고리즘, 리소스, 컨펌, 등록되었습니다~~
→ 전부 "가장 좋아요 / 모일 수 있어요 / 넣었어요"의 사람 말로.
