# UX 라이팅 시스템

토스 라이팅 문법을 이 제품에 맞게 번역한 5원칙과, 화면별 적용입니다.

## 원칙

1. **버튼은 결과를 미리 말한다** — "확인" 대신 "추천 시간 보기", "목요일 오전 10시로 잡기". 누르기 전에 무슨 일이 생길지 안다.
2. **행동의 결과를 즉시 되돌려준다** — 보내기 → 토스트 "응답을 보냈어요". 시스템이 받았음을 사람 말로 확인.
3. **시스템 용어 대신 사람 말** — 필수/선택 → "꼭 와야 해요 / 안 와도 돼요". 이 말투가 만들기→추천→확정까지 일관되게 이어진다.
4. **누구의 잘못도 아니게** — "불참 ❌"이 아니라 "선택 참석이라, 외근이어도 회의는 그대로 진행돼요". 사실은 투명하게, 죄책감은 0으로.
5. **과장하지 않는다** — "6명 전원 확정!"이 아니라 "박지후님이 조정하면 6명 전원이 모여요". 조건은 조건문으로. 신뢰는 정확함에서 온다.

## 화면별 적용

| 화면 | 문구 | 원칙 |
|---|---|---|
| 만들기 | "누가 모여야 하나요?" (기능명 아닌 질문) | 3 |
| 만들기 | "나(김화연)는 자동으로 포함돼요" | 2 |
| 조율 | "일정 제목은 안 보고, 비어 있는지만 확인했어요" | 5 (프라이버시 안심) |
| 조율 | "다음 주까지 · 4일 남았어요" | 1 (마감 인지) |
| 동료 탭 | "안 되는 시간만 알려주세요 · 다 괜찮으면 그냥 보내기" | 1, 3 |
| 추천 | "이 시간이 가장 좋아요" + 근거 한 줄 | 1, 5 |
| 추천 | "강수아님은 선택 참석이라, 외근이어도 그대로 진행돼요" | 4 |
| 조정 부탁 | "어려우면 목요일 10시로 진행할게요!" (상대에게 출구) | 4 |
| 대기 | "어느 쪽이든 회의는 화요일 11시예요" | 5 (불안 제거) |
| 대기 | "급하지 않으면 그냥 기다려도 괜찮아요" | 4 (행동 강요 없음) |
| 확정 | "6명 캘린더에 모두 넣었어요 / 다들 알림을 받았어요" | 2 |
| 변경 | "4명이 와요 · 지금보다 1명 줄어요" (손해도 정직하게) | 5 |
| 변경 확정 | "박지후님이 조정하면 6명 전원이 모여요" (조건문) | 5 |

## 금지어

계산·시스템 냄새가 나는 말: ~~가중치, 최적화, 교집합, 알고리즘, 리소스, 컨펌~~
→ 전부 "가장 좋아요 / 모일 수 있어요 / 괜찮아요"의 사람 말로.
