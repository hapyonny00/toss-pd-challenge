# oneui — 회의 조율 서비스 · One UI 최종 명세서 + 프로토타입

토스 2026 PD 챌린지의 **삼성 One UI 8.5 버전** 산출물. 기존 `gcal-prototype`(데스크톱 웹)·v3(TDS mobile)와 별개로, "이동 중에도 회의를 잡는 모바일 네이티브" 방향을 One UI의 하단 집중 구조 위에 설계했다.

## 무엇이 들어있나
- **[SPEC.md](SPEC.md)** — UI 최종 명세서. 개발자·디자이너가 이 문서만으로 착수 가능한 수준.
  토큰(라이트/다크)·컴포넌트 12종·화면별 IA/픽셀/상태/카피·자동 태깅 엔진·엣지케이스·마이크로카피 전문·접근성.
- **[prototype.html](prototype.html)** — 렌더 프로토타입. 브라우저로 바로 열면 됨(빌드 없음).
  상단 컨트롤로 **화면(홈/참석자/히트맵/추천/완료/타협) × 모드(라이트/다크)** 토글. 딥링크 `?stage=heatmap&theme=dark`.
- **screens/** — 헤드리스 크롬 검증 스크린샷 6종(라이트/다크 혼합).

## 설계 핵심 3줄
1. **하단 집중 구조** — 화면 위=보는 영역(거대 타이틀+캘린더), 아래=만지는 영역(바텀시트+주 버튼). 한 손 조작.
2. **히트맵 3겹 위계** — 개인 6색을 버리고 면(적합도)·선(추천)·점(부담)으로 분리. 파란 테두리는 화면에 딱 3곳.
3. **막다른 길 금지** — 6명 모두 되는 시간 0개일 때 "없습니다" 대신 타협 카드 3장(제외+요약 / 기피+양해 대필 / 30분 단축).

## 검증
헤드리스 크롬으로 6개 화면 × 라이트/다크 렌더 확인 완료:
```sh
"/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
  --headless=new --window-size=460,1040 --screenshot=out.png \
  "file://$PWD/prototype.html?stage=heatmap&theme=dark"
```
