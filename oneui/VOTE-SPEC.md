## 0. 시나리오 요약과 설계 판단 3개

**시나리오**: 주최자가 회의를 만들면 팀원 스케줄이 캘린더에 오버레이된다. 전원 일치 시간이 없으면 우측 패널에 **AI 추천 카드**(1~2명만 조정하면 되는 시간)가 생성되고, 주최자가 카드를 캘린더로 **드래그해 스냅**하면 해당 팀원들에게 **3단계 퀵 투표**가 발송된다. 투표 현황은 카드 안에 칸반 진척도 스타일로 실시간 반영된다.

이 문서 전체를 관통하는 설계 판단:

| # | 판단 | 근거 |
|---|---|---|
| D1 | **드롭 = 발송** (확인 모달 없음, 5초 실행취소로 대체) | 스냅 인터랙션의 가치는 "가설을 가볍게 놓아보는 감각". 모달은 그 가벼움을 죽인다 |
| D2 | **중간 옵션이 전략의 핵심** | '불가' 대신 "+30분이면 가능"이라는 연속 데이터를 수거 → 재조율 라운드가 아니라 마이크로 조정 1회로 수렴 |
| D3 | **감정 비용은 시스템이 흡수** | 거절 버튼에 사과를 내장하고, 전달 문구는 AI가 '사실+대안' 구조로 재포맷 |

## 1. One UI 화면 설계 & 컴포넌트 매핑

### 1.1 One UI Design Kit 에셋 매핑표

| 우리 컴포넌트 | One UI Kit 에셋 | 변형 포인트 |
|---|---|---|
| 좌측 내비 (데스크톱) | `Navigation Rail` | 72px 아이콘 레일, 선택 항목 pill 하이라이트 |
| 상단 바 | `Top app bar` (collapsing) | 거대 타이틀 28px → 스크롤 시 20px 붕괴 |
| AI 추천 카드 | `Card` (Focus Block) | radius 26 → **20**(사이드바 폭 340 기준 시각 보정) |
| 추천 근거 배지 | `Large chips` | 앰버 틴트, 좌측 ✨ 아이콘 |
| 타임라인 셀 | 커스텀 (Kit에 없음) | §1.4에서 신규 정의 |
| 3단계 투표 버튼 | `Buttons` (Filled/Tonal) | 세로 스택 52px, §3 |
| 오프셋 조절 휠 | `Segmented sliders` + `Slide navigation` | 프리셋 칩 4개 + 커스텀 스크럽 |
| 투표 바텀시트 | `Bottom container` | grabber 32×4, radius 상단 26 |
| 발송 토스트 | `Toast` | 하단 96px 위, 실행취소 액션 포함 |
| 투표 진척 바 | 커스텀 (weihu 칸반 도트 차용) | §3.3에서 신규 정의 |

### 1.2 데스크톱 대시보드 블루프린트 (≥1280px)

```
┌─Rail 72─┬──────── Main (fluid, min 720) ────────┬── Aside 340 ──┐
│ ◐ 로고   │ TopAppBar h64                           │ pad 24        │
│ ▣ 홈     │  [오늘] ‹ › ── "7월 13–17 · 다음 주"      │ ✨ AI 추천 (3) │
│ ▣ 캘린더* │ ────────────────────────────────    │ ┌───────────┐ │
│ ▣ 회의   │ WeekGrid: 56px 시간축 + 5col            │ │ RecoCard#1 │ │
│ ▣ 팀     │  셀 h56 · hairline #ECECEE              │ │ (§1.3)     │ │
│          │  이벤트 필 radius 10                     │ ├───────────┤ │
│ (하단)    │  투표중 카드: 점선 테두리 (§1.4)           │ │ RecoCard#2 │ │
│ ⚙ 설정   │ 하단 좌: 레이어 토글 + 범례                │ │ RecoCard#3 │ │
└─────────┴───────────────────────────────────┴───────────────┘
```

- **세이프존**: Aside 내부 패딩 24, 카드 간 12, Main 좌우 24. 모든 콘텐츠 블록은 Focus Block(라운드 컨테이너)으로 묶고 블록 사이는 배경색(`surface/base`)이 보이게 한다 — One UI의 "블록 사이 숨 쉬는 배경" 원칙.
- Aside 헤더: `✨ AI 추천` 15px/700 + 카운트 배지(`accent/fill` 원형 20px). 그 아래 서브 카피 12px `text/secondary`: *"모두 되는 시간이 없어서, 조금만 조정하면 되는 시간을 찾았어요."*

### 1.3 AI 추천 카드 (RecoCard) — 정밀 스펙

```
┌─ RecoCard (w 292 = 340-24·2, h auto) ──────────┐
│ pad 16                                          │
│ ✨ [준호 님 하나만 조정하면 돼요]   ← Chip h28      │
│ 화요일 오후 2시                    ← 17px/700     │
│ 1시간 · 팀 킥오프                  ← 13px/500 sec │
│ ●●●●○○  4/6 답했어요             ← DotBar (§3.3)│
│ (지✓)(준~)(서✓)(민✓)(도·)(하·)    ← AvatarChips  │
│ ┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄  ← 조정 제안 시    │
│ 💬 준호: 30분 뒤면 가능  [30분 뒤로 옮기기]         │
└─────────────────────────────────────────────┘
```

| 속성 | Light | Dark |
|---|---|---|
| 배경 | `surface/1` #FFFFFF | `surface/1` #1B1B1D |
| radius | **20px** | 동일 |
| 테두리 | 없음 (표면 명도로 분리) | 없음 |
| 근거 칩 | bg `Amber50` #FFF3DC · text `Amber900` #5A3600 | bg #3A2E14 · text `Amber400` #FFB43A |
| 드래그 핸들 힌트 | hover 시 `cursor:grab` + 배경 `surface/2` | 동일 |
| pending(캘린더 안착 후 사이드바 잔상) | 1.5px dashed `Gray200` placeholder | dashed `Gray800` |

### 1.4 타임라인 셀 & 캘린더 내 투표 카드

| 요소 | 값 |
|---|---|
| 셀 | h56 · hairline `Gray100`(라이트) / `Gray800`(다크) · radius 0 |
| 유효 드롭 존 점등 | bg `Blue50` / 다크 `#12233F` · 전환 200ms ease-out |
| 무효 영역 | 기존 콘텐츠 채도 -40% (filter: saturate(.6) opacity(.55)) |
| 스냅 프리뷰 고스트 | 카드가 셀 크기로 morph · bg `Blue100` 60% · 1.5px dashed `accent/fill` |
| 안착 카드(투표 중) | bg `Blue50`/`#12233F` · **1.5px dashed** `accent/fill` · radius 10 · 내부에 타이틀 11.5px + 미니 도트바 |
| 확정 카드 | dashed → **solid** · bg `accent/fill` #387AFF · 텍스트 white · radius 10 |

> 점선=아직 약속 아님, 실선=약속. 이 한 가지 시각 문법으로 상태를 설명 없이 전달한다.

### 1.5 모바일 앱 뷰 블루프린트 (<600dp)

```
┌────────── 360dp ──────────┐
│ StatusBar                  │
│ TopAppBar: "7월" 거대타이틀   │ ← Viewing area
│ WeekStrip: 월13 ~ 금17      │
│ ┌────────────────────┐    │
│ │ 세로 타임라인 (셀 h64)   │    │
│ │ ▒투표중 카드(점선)▒     │    │
│ └────────────────────┘    │
│ ╭─ BottomSheet peek 96 ─╮ │ ← Interaction area
│ │ ⠿  ✨ 추천 3개          │ │
│ │ [Card1][Card2][Card3]→ │ │  가로 캐러셀 (snap)
│ ╰───────────────────────╯ │
│ NavBar (홈·캘린더·회의·팀)    │
└───────────────────────────┘
```

- 시트 3단계: **peek 96dp**(핸들+헤더) → **half 55%**(카드 풀 아나토미) → **full 92%**(카드 + 근거 상세). 시트 radius 상단 26, grabber 32×4 `Gray200`.
- 마진: 좌우 16dp(모바일) / 24dp(태블릿+). 카드 풀너비-32.
- 주 버튼(여기로 잡기/투표 확정)은 항상 시트 최하단 고정, h52, radius 26(풀라운드).

## 2. 드래그&드롭 · 자석 스냅 인터랙션 명세

### 2.1 데스크톱 — 상태 머신과 Smart Animate 값

| 단계 | 트리거 | 시각 반응 | Figma 세팅 |
|---|---|---|---|
| **A. Idle→Lift** | 카드 pointerdown 120ms 유지 | scale 1→1.03, 그림자 0→`0 12px 28px rgba(0,0,0,.16)`, 원위치에 점선 placeholder | Smart Animate · Ease Out · **160ms** |
| **B. Dragging** | 포인터 이동 | 카드가 커서 추적(중심 오프셋 유지). 캘린더: 유효 슬롯 `Blue50` 점등, 무효 채도↓. Aside 딤 12% | While Dragging(프로토타입은 On Drag 트리거 + 오버레이 근사) |
| **C. Near-snap** | 유효 슬롯 중심 반경 **30px** 진입 | 고스트가 슬롯 크기로 morph되며 흡착 프리뷰. 슬롯 테두리 1.5px `accent/fill` | Smart Animate · **Ease In-Out · 200ms** |
| **D. Snapped** | pointerup | 카드가 셀로 스프링 안착(overshoot 1.02→1.0), 셀 플래시 1회, 카드 상태 `pending` 전환, Toast *"세 분에게 물어볼게요 · 실행 취소"* 5s | Smart Animate · **Spring(Bouncy, damping .7) 또는 Ease-Out-Back 320ms** |
| **E. Invalid drop** | 무효 영역 pointerup | 카드가 사이드바 원위치로 복귀 | Smart Animate · Ease In-Out · **360ms** |
| **F. Cancel** | ESC / 우클릭 | E와 동일 복귀 | 동일 |

**보조 경로(비드래그)**: 카드 클릭 → 대상 슬롯 하이라이트 + 자동 스크롤 + 슬롯 위 [여기로 잡기] 버튼. 접근성(키보드: Enter로 pick, 화살표로 슬롯 이동, Enter로 drop) 겸용.

### 2.2 모바일 — 1:1 대응 변환 원리

데스크톱의 공간 이동(사이드바→캘린더)을 모바일에서는 **수직 이동(시트→타임라인)**으로 보존한다. 구조가 따로 놀지 않는 이유는 두 뷰 모두 "**행동 영역에서 판단 영역으로 카드를 올려보낸다**"는 같은 은유를 쓰기 때문.

| 데스크톱 | 모바일 대응 | Figma 세팅 |
|---|---|---|
| 드래그 시작 | **롱프레스 250ms** → 햅틱 light → 시트가 40% 투명으로 접히고 타임라인이 드롭 존으로 전환 | On Long Press → 오버레이 |
| 드래그 중 자동 이동 | 화면 상/하단 80dp 진입 시 타임라인 자동 스크롤 | (구현 노트로 전달) |
| 스냅 | 동일 — 슬롯 반경 24dp 흡착 | Smart Animate 200ms |
| **기본 경로 (드래그보다 우선)** | **탭-투-스냅**: 카드 탭 → 타임라인이 해당 슬롯으로 자동 스크롤 + 고스트 프리뷰 안착 → 시트 최하단 [여기로 잡기 h52] 확정 | On Tap → Scroll To + Smart Animate 240ms |

> 모바일 기본은 탭-투-스냅이다. 세로 스크롤과 드래그 제스처가 충돌하는 폼팩터에서 D&D를 기본으로 강제하는 건 데스크톱 은유의 과잉 이식. '이동' 은유는 자동 스크롤+고스트 morph로 보존된다.

## 3. 3단계 투표 & 피드백 컴포넌트 설계

### 3.1 투표 버튼 세트 — Variants

수신자 화면(모바일 바텀시트 기준. 데스크톱은 카드 내 가로 3버튼):

```
회의 제안이 왔어요 — 화요일 오후 2시 · 1시간
지민 님이 팀 킥오프를 잡고 있어요.

[ 가능해요 😊 ]              ← Filled · accent/fill · h52 · r26
[ 조금만 조정하면 돼요 🤔 ]     ← Tonal · Blue50/text Blue700 · h52
[ 어려워요, 미안! 🙏 ]         ← Text button · text/secondary · h44
```

**Figma Component: `VoteSheet`** — property `state`:

| state | 시각 |
|---|---|
| `idle` | 3버튼 기본 |
| `picked-yes` | 가능 버튼만 남고 체크 morph → 시트 자동 닫힘 400ms |
| `picked-adjust` | **시트가 아래로 확장**되며 오프셋 스텝 노출 (§3.2) |
| `picked-no` | 완충 카피 1줄 표시 후 닫힘: *"전할게요. 미안해할 일 아니에요 — 다른 시간을 이미 찾고 있어요."* |

버튼 위계 의도: 긍정 응답이 Filled(가장 크고 진함), 중간이 Tonal, 거절이 Text — **거절을 못 하게 숨기는 게 아니라, 시각 위계로 "거절해도 작은 일"이라는 메시지**를 준다.

### 3.2 '조금만 조정하면 돼요' — 오프셋 마이크로 인터랙션

`picked-adjust` 시 시트 높이 320→460 확장(Smart Animate, Ease In-Out 280ms), Interaction area에 노출:

```
얼마나 옮기면 편해요?
[15분 늦게] [30분 늦게] [1시간 뒤] [점심 이후로]   ← Chips h36 r18, 다중선택 불가
── 또는 ──
‹  ──────●──────  ›   -1h … +2h              ← 오프셋 스크럽 휠 (15분 단위 스냅, 햅틱 틱)
✏️ "예: 2시 이후면 언제든 좋아요"                 ← 자유 입력 (선택)
[ 이 조건으로 보내기 ]                          ← Filled h52
이 내용은 지민 님에게만 보여요. 옮길지는 지민 님이 정해요.
```

**타임라인 미세 이동(Offset) 그래픽 연출** — 주최자가 조정 제안을 수락하면:
1. 캘린더의 점선 카드가 원위치에 **고스트 잔상**을 남기고
2. 카드 본체가 오프셋만큼 **수직 슬라이드**(Smart Animate 320ms Ease In-Out — 30분=셀 절반 28px)
3. 이동 경로에 2px `accent/fill` 세로 가이드라인이 0.4s 표시됐다 페이드
4. 잔상이 사라지며 카드에 `조정됨 +30분` 미니 칩이 3초 표시
5. 조정 영향자에게만 재투표 발송 (전체 재발송 금지 — 라운드 증식 방지)

### 3.3 투표 진척 바 (DotBar) — 칸반 진척도의 One UI 번역

weihu 레퍼런스의 도트 프로그레스를 One UI 문법(플랫, 곡률, 명도 분리)으로:

```
●●●●○○   4/6 답했어요
```

| 속성 | 값 |
|---|---|
| 도트 | 10×10 원, 간격 6, 참석자 수 = 도트 수 (최대 8, 초과 시 바 형태 전환) |
| 채움 색 | 가능 `Green400` · 조정 `Amber400` · 불가 `Red400` · 미응답 `Gray200`(다크 `Gray800`) 빈 원 |
| 채움 순서 | 응답 도착순 좌→우. 도트 채움 애니메이션: scale 0.6→1 + fade, 180ms |
| 카운트 라벨 | 12px `text/secondary` "4/6 답했어요" — 숫자만 강조 500 |
| AvatarChips | 28px 원 + 우하단 상태 배지 12px(✓ Green / ~ Amber / ✕ Red / 미응답 점선 외곽) · 응답 시 **flip 애니메이션 rotateY 180° 200ms** |

**카드 상태 Variants: `RecoCard.state`**
`idle | dragging | pending | voting | adjust-suggested | confirmed | returned`
- `confirmed`: 점선→실선, DotBar가 단색 Green 채움으로 통합, 체크 morph. 컨페티 금지 — 이 제품의 톤은 "당연히 되는 일이 조용히 됨".
- `returned`(과반 불가): 카드가 캘린더에서 사이드바로 자동 복귀 + 재계산. **막다른 길 금지.**

## 4. 심리적 넛지 UX 라이팅 가이드북

### 4.1 요청자(주최자) 시점 — 강요를 덜어낸 제안 문구

| 시점 | 카피 | 원칙 |
|---|---|---|
| 드롭 직후 토스트 | "세 분에게 물어볼게요" [실행 취소] | '통보'가 아니라 '질문'임을 동사로 못박음 |
| 조정 수락 버튼 | "30분 뒤로 옮기기" | 명령형 최소 동사. "요청" "승인" 같은 결재 언어 금지 |
| 조정 후 전체 알림 | "회의가 30분 뒤로 옮겨졌어요. 준호 님 의견을 반영했어요 — 캘린더에 업데이트해 뒀어요." | 기여자 명시로 '의견 낸 보람' 반환 + 확정=등록 |
| 리마인드 발송 확인 | "도윤 님에게 한 번 더 물어볼까요? 버튼 하나면 답할 수 있게 보낼게요." | 재촉을 '쉬움 강조'로 반전 |

### 4.2 수신자 시점 — 위트 있는 대안 카피 3세트

| 세트 | 가능 | 조정 | 불가 |
|---|---|---|---|
| **기본** (범용) | 가능해요 😊 | 조금만 조정하면 돼요 🤔 | 어려워요, 미안! 🙏 |
| **A. 스쿼드** | 콜! 바로 갈게요 🙌 | 살짝만 밀면 가능 ⏱ | 이번엔 도저히 🙈 |
| **B. 위트** | 도장 쾅 ✅ | 협상 가능해요 🤝 | 패스… 다음엔 꼭! 🥲 |
| **C. 미니멀** | 돼요 | 조건 있어요 | 안 돼요 (+ 시스템이 완충 문구 자동 부착) |

작성 규칙: ① 8자 내외(세로 스택 줄바꿈 금지) ② 이모지 끝 1개 ③ **거절 버튼에 사과를 내장**해 누르는 사람의 죄책감을 버튼이 대신 지불 ④ 팀 설정에서 세트 선택(조직 문화 적응형).

### 4.3 거절 전달 — AI 자동 재포맷 넛지

원문 그대로 전달하지 않는다. **사실 + 대안** 구조로 재포맷:

| 수신자가 남긴 것 | 주최자에게 전달되는 문장 |
|---|---|
| [불가] (사유 없음) | "준호 님이 이 시간은 어렵대요. 대신 목요일 오후가 비어 있어요." |
| [불가] + "그날 데드라인이라…" | "준호 님이 이날은 마감이 있어서 어렵대요. 금요일 오전이라면 모두 가능해요." |
| [조정] +30분 | "준호 님이 30분 뒤면 가능하대요. 한 번에 옮길까요?" [30분 뒤로 옮기기] |

재포맷 3원칙: **사과 문구를 넣지 않는다**(넣는 순간 거절이 '잘못'이 된다) · **대안을 반드시 동반한다**(시스템이 이미 다음 수를 찾았음을 보여 안심) · **감정 표현은 수신자가 고른 이모지 1개까지만 통과**.

## 5. Figma 핸드오프 노트

1. **레이어 네이밍** — 카드가 사이드바→캘린더→시트를 넘나들며 morph하므로 세 프레임에서 동일 레이어명(`card-reco-1`) 유지. Smart Animate 매칭 필수 조건.
2. **Variants 매트릭스** — `RecoCard`: state(7) × size(desktop/mobile) × theme(light/dark) = 28. 도트바·아바타칩을 중첩 컴포넌트로 분리해 조합 폭발 방지.
3. **드래그 근사** — Figma는 free-drag 시뮬레이션이 약함. `On Drag` 트리거 + near-snap은 variant 전환으로 근사하고, 실제 감각 검증은 [`prototype-vote.html`](prototype-vote.html)로 한다(코드가 곧 모션 스펙).
4. **다크모드** — 토큰 스왑만으로 성립해야 함. raw hex 직접 참조 금지, 전 컴포넌트 `surface/*` `text/*` `accent/*` 시맨틱 토큰만.
5. **접근성** — 투표 3버튼 최소 터치 타깃 48dp, 도트바에 aria-label "6명 중 4명 응답, 3명 가능, 1명 조정", 드래그의 키보드 대체 경로(§2.1 보조 경로) 필수.
